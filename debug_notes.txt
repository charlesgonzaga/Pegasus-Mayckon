BUG ENCONTRADO:
O erro é na query topClientes quando filtro de mês está ativo.
A query usa: AND DATE_FORMAT(`notas`.`dataEmissao`, '%Y-%m') = ?
Mas o problema é que Drizzle adiciona backticks ao redor de `notas`.`dataEmissao` 
dentro do sql template literal, o que causa conflito com a referência de tabela 'n' usada no raw SQL.

A query usa alias 'n' para notas: FROM notas n
Mas o filtro de mês usa: DATE_FORMAT(`notas`.`dataEmissao`, '%Y-%m')
Deveria usar: DATE_FORMAT(n.dataEmissao, '%Y-%m')

O mesmo problema existe no mesFilterTop na linha 572 do db.ts:
const mesFilterTop = mes ? sql`AND DATE_FORMAT(${notas.dataEmissao}, '%Y-%m') = ${mes}` : sql``;

Drizzle converte ${notas.dataEmissao} para `notas`.`dataEmissao` mas a query usa alias 'n'.
Solução: usar raw SQL string em vez de referência Drizzle.
